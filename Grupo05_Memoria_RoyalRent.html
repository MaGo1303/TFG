1. Introducción

RoyalRent es una aplicación web desarrollada como Proyecto de Fin de Ciclo del Grado Superior en Desarrollo de Aplicaciones Multiplataforma (DAM). Se trata de una plataforma pensada para el alquiler de vehículos de lujo —coches, yates y helicópteros— que permite a los usuarios explorar un catálogo completo, gestionar un carrito de compra, registrarse, iniciar sesión y realizar reservas de forma sencilla y ágil.

La aplicación se compone de dos partes bien diferenciadas. El frontend es una aplicación SPA (Single Page Application) construida con React, que ofrece una interfaz moderna, adaptable a cualquier dispositivo y con animaciones cuidadas. El backend, por su parte, es una API REST desarrollada con Node.js y Express, que se encarga de gestionar usuarios, productos, pedidos y la autenticación mediante tokens JWT.

El catálogo incluye veinte coches de marcas como Ferrari, Lamborghini o Rolls-Royce; doce yates de astilleros como Azimut, Sunseeker o Ferretti; y ocho helicópteros de fabricantes como Bell, Airbus o AgustaWestland. Todos ellos cuentan con descripciones detalladas, precio por día e imágenes representativas. El sistema cuenta además con un panel de administración desde el que los gestores pueden supervisar usuarios, productos, pedidos y mensajes de contacto.

1.1. Motivación

La idea de RoyalRent nace de la necesidad de digitalizar el proceso de alquiler de vehículos de alta gama, un sector que tradicionalmente ha dependido de procesos manuales y poco eficientes. Una plataforma digital permite agilizar las reservas, mejorar la experiencia del cliente y ofrecer herramientas de gestión a los administradores del negocio.

Además, este proyecto permite poner en práctica y consolidar los conocimientos adquiridos durante el ciclo formativo, combinando programación, bases de datos, desarrollo web y diseño de interfaces en un único proyecto completo.

1.2. Objetivos

Los objetivos que se persiguen con este proyecto son los siguientes:

  • Diseñar y desarrollar una aplicación web completa y funcional, desde la base de datos hasta la interfaz de usuario.
  • Implementar un sistema de autenticación seguro basado en tokens JWT con contraseñas protegidas mediante bcrypt.
  • Crear una API REST para la gestión de usuarios, productos y pedidos.
  • Diseñar una base de datos relacional que refleje correctamente las entidades del sistema y sus relaciones.
  • Desarrollar una interfaz de usuario clara, moderna y adaptable a cualquier dispositivo.
  • Aplicar buenas prácticas de desarrollo de software, incluyendo control de versiones y documentación.

Enfoque del proyecto. RoyalRent se enmarca dentro de un proyecto de desarrollo aplicado: la creación de una aplicación funcional relacionada con el campo de estudio del alumno. El proyecto se apoya en situaciones reales y se organiza siguiendo un plan de trabajo por fases.

Relaciones entre las tablas

Las tablas se relacionan así:

  • users → orders: un usuario puede hacer varios pedidos (1:N).
  • orders → order_items: un pedido puede tener varias líneas (1:N).
  • items → order_items: un vehículo puede estar en varios pedidos (1:N).
  • items → item_images: un vehículo puede tener varias imágenes (1:N).

En la tabla item_images se ha definido el borrado en cascada, de forma que al eliminar un vehículo se borran automáticamente todas sus imágenes asociadas.

7.4. Diseño de las interfaces

El diseño de RoyalRent busca ser elegante y funcional. La paleta de colores combina tonos oscuros con detalles en dorado como color de acento. Estas son las pantallas principales:

Página principal. Comienza con un hero que muestra un vídeo de fondo corporativo. Sobre él aparece el título de la web y un botón que lleva al catálogo. Más abajo hay estadísticas animadas, reseñas de clientes con efecto marquee infinito, tres tarjetas de servicios con efecto 3D, la presentación del equipo y un formulario de contacto funcional.

Catálogo de servicios. Los vehículos se muestran en una cuadrícula con pestañas para filtrar por categoría. Cada tarjeta incluye imagen, nombre, precio y disponibilidad. Al hacer clic se abre un panel lateral con la información completa del vehículo y un carrusel de imágenes.

Autenticación. Una sola página que combina inicio de sesión y registro, alternando entre ambos con una animación. Incluye validación de campos, mensajes de error claros y un indicador de carga.

Carrito y pago. El proceso va en dos pasos: primero se revisan los productos seleccionados y después se simula el pago. Se puede elegir entre varias tarjetas y se muestra un resumen con el total.

Perfil de usuario. Tiene dos columnas: a la izquierda un formulario para actualizar los datos personales, y a la derecha el historial de reservas, que se puede expandir para ver más detalles.

Panel de administración. Muestra estadísticas generales, gráficos por categorías, listado de pedidos recientes y tablas para gestionar usuarios, productos y mensajes.

7.6. Funcionalidad del proyecto

Sistema de autenticación

RoyalRent usa JSON Web Tokens (JWT) para la autenticación. Cuando un usuario se registra, el servidor valida los datos, cifra la contraseña con bcrypt (10 rondas) y la guarda en la base de datos. Al iniciar sesión, el servidor comprueba la contraseña y, si es correcta, genera un token con validez de 24 horas. Ese token se almacena en el navegador (localStorage) y se envía en cada petición mediante la cabecera Authorization. Un middleware se encarga de verificar el token en las rutas protegidas; si falta o es inválido, devuelve un error 401 o 403.

Catálogo

El catálogo muestra todos los vehículos y permite filtrarlos por tipo: coches, yates o helicópteros. Cada vehículo tiene varias imágenes en un carrusel, una descripción, especificaciones y el precio por día. Los administradores pueden añadir, editar y eliminar vehículos, y también cambiar su disponibilidad.

Reservas

El proceso de reserva tiene dos pasos: primero el usuario añade vehículos al carrito (que se guarda en localStorage para no perderse al cerrar el navegador), y después completa el pago simulado. Al confirmar, el servidor crea el pedido dentro de una transacción SQL: si algo falla, se deshace todo y los datos quedan intactos.

Panel de administración

El panel ofrece estadísticas (usuarios, pedidos, ingresos), gráficos por categorías, y tablas para gestionar usuarios, productos, pedidos y mensajes de contacto.

8. Conclusiones y Mejoras

8.1. Conclusiones

Una vez terminado el proyecto, se pueden destacar estos puntos:

  • Se ha conseguido una aplicación web completa y funcional que cumple los requisitos planteados en el anteproyecto.
  • La separación entre frontend y backend facilita el mantenimiento y permite reutilizar el backend con otras aplicaciones.
  • La autenticación con JWT ofrece un mecanismo de seguridad sólido y adecuado para una API REST.
  • El diseño responsivo garantiza una buena experiencia tanto en móvil como en escritorio.
  • Las transacciones SQL aseguran que los pedidos se creen de forma íntegra, sin riesgo de datos inconsistentes.
  • Se han aplicado conocimientos de los ocho módulos del ciclo, lo que ha permitido consolidar el aprendizaje en un proyecto real.

8.2. Mejoras propuestas

De cara al futuro, se podrían incorporar las siguientes mejoras:

  • Pago real: integrar Stripe o PayPal para procesar pagos reales.
  • Notificaciones: enviar correos de confirmación de reserva (el sistema nodemailer ya está configurado).
  • Calendario: añadir un calendario interactivo para elegir fechas y ver disponibilidad en tiempo real.
  • Búsqueda avanzada: filtrar por precio, marca, modelo o características.
  • Valoraciones: permitir a los usuarios puntuar los vehículos que han alquilado.
  • Tests automáticos: incorporar pruebas unitarias y de integración.

10. Anexos

Anexo A: Guía de instalación

Prerrequisitos

  • Node.js (v18 o superior).
  • MySQL 8.0 (XAMPP o similar).
  • npm (incluido con Node.js).

Pasos

  1. Arrancar MySQL (XAMPP → Start MySQL).
  2. Crear el archivo .env en backend/: 
    DB_HOST=localhost
DB_USER=root
DB_PASSWORD=
DB_NAME=royalrent
JWT_SECRET=royalrent_secret_key
PORT=5000
  3. Inicializar la base de datos: 
    cd backend
npm install
npm run init-db
  4. Iniciar el backend: 
    cd backend
npm start
  5. Iniciar el frontend: 
    cd frontend
npm install
npm run dev

La web se abre en http://localhost:5173 y la API en http://localhost:5000.
